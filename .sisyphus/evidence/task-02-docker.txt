## Configuration Details

### postgres-app Service
- Image: `postgres:16-alpine`
- Health check: `pg_isready -U pisang -d pisangdb`
- Volume: `pgdata` for persistent data
- Resource limits: 0.5 CPU, 768M RAM

### postgres-sandbox Service
- Image: `postgres:16-alpine`
- Health check: `pg_isready`
- Volume: `pgsandbox` for persistent data
- Resource limits: 0.5 CPU, 768M RAM
- Port: 5433 (mapped from container 5432)

### app Service
- Build context: `.`
- Target: `dev` (development stage)
- Depends on: Both postgres services with health check conditions
- Resource limits: 1.0 CPU, 512M RAM
- Volumes: Source code bind mount with node_modules preservation

### Network
- Name: `pisangdb-network`
- Driver: `bridge`
- All services connected for inter-service communication
